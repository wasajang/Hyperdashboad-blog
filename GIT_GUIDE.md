# Git 초보자를 위한 작업 가이드

이 가이드는 `Hyperdashboad-blog` 프로젝트를 작업할 때 필요한 기본적인 Git 명령어들을 설명합니다.

## 1. 작업 시작 전: 최신 코드 가져오기 (Pull)
작업을 시작하기 전에 항상 원격 저장소(GitHub)에 있는 최신 변경사항을 내 컴퓨터로 가져와야 합니다. 다른 PC에서 작업한 내용이 있을 수 있기 때문입니다.

```bash
git pull origin main
```
*참고: `main`은 기본 브랜치 이름입니다. 만약 다른 브랜치를 사용한다면 그 이름을 써야 합니다.*

## 2. 작업 진행
코드를 수정하거나 새로운 파일을 만듭니다.

## 3. 변경사항 저장 준비 (Add)
작업한 파일들을 저장(커밋)할 준비를 합니다.

* **모든 변경사항 추가하기 (가장 많이 사용)**
```bash
git add .
```

* **특정 파일만 추가하기**
```bash
git add 파일경로/파일이름
```

## 4. 변경사항 확정 (Commit)
준비된 변경사항들에 설명을 붙여서 확정합니다. 메시지는 작업 내용을 명확하게 적는 것이 좋습니다.

```bash
git commit -m "작업한 내용에 대한 간략한 설명 (예: 메인 페이지 로고 수정)"
```

## 5. GitHub에 올리기 (Push)
확정된 변경사항(커밋)을 GitHub 원격 저장소로 보냅니다.

```bash
git push origin main
```

---

## 요약: 작업 순서
1. **`git pull`** (시작할 때)
2. 코드 수정
3. **`git add .`**
4. **`git commit -m "메시지"`**
5. **`git push`** (작업 끝날 때)

## 상태 확인 (Status)
현재 어떤 파일이 수정되었는지, 어떤 상태인지 궁금할 때는 언제든 다음 명령어를 입력하세요.

```bash
git status
```
